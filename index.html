Overview

Welcome to my React development project for CSCI1300! I analyzed the design and functionality of various online platforms and drew inspiration from this to create a functional aggregator interface with React. This application allows users to sort, filter, and select items from a list – this concept is similar to that of an online shopping cart. Below, you will find a competitive analysis of three websites as well as a narrative process of my work in this project.

Part 1. Competitive Analysis

First, I chose 3 different websites with a favoriting function: Amazon Wishlist , Goodreads, and Pinterest. Here, I examined how they organized information that is relevant to the user and how aggregators are used. (Aggregrators are basically lists that keep track of items).

The analysis of each website's favoriting function shows us some key insights. Amazon Wishlist has great organization — its filtering options are intuitive and relevant without adding too many unnecessary options. However, the visual aspect of the page is a bit lacking in terms of aesthetic customization. While Goodreads offers a high level of customization and filtering options with its organization, bookshelves aren't collaborative and lack privacy options, which does reinforce the fact that it's intended to be a social app, but this could possibly deter users who prefer more privacy. Finally, Pinterest stands out in visual appeal and intuitiveness, which makes it a great choice for visual planning, though the organization system could be streamlined to avoid unnecessary complexity. In summary, each platform has its strengths, but could all be improved from balancing advanced features with simplicity to cater to a broader audience.

Based on factors and designs that worked well for these websites, I came up with a list of features that I would like to implement. Intuitive filtering and organization is crucial, so inspired by Amazon Wishlist, I'd like to implement a filtering system based on relevance and easy navigation. There should be categories/tags used for organization which allow users to quickly find items. Also, I want to ensure that the organization remains simple so that users can easily manage their lists without feeling overwhelmed. Finally, the list should be compact and efficient so that users can scan their lists without unnecessary clutter.

Part 2. Implementation

For this course, we use Vercel rather than Github pages to deploy React apps from scratch.

Before getting started on coding, I planned out my interface, which would be centered around the theme of culinary recipes. This app is intended for cooking enthusiasts who want to explore and organize recipes. The aggregator section is called "My Recipe Book" which allows users to save their favorite recipes. This is an important part of the app, designed to show saved recipes with their names, cuisines, and dietary restrictions. It also updates the total number of recipes saved in real-time. Features included are as follows:

  • 2 filtering categories: Cuisine (Italian, Japanese, Chinese, etc) and dietary restrictions (vegetarian, vegan, gluten-free)
  • 1 sorting feature: Preparation time for each recipe
  • 1 Aggregator: "My Recipe Book" which displays recipes with the name, cuisine, and dietary restriction/preference. This should also dynamically update the total count of recipes in the book.
  • 12 Item cards: Each recipe card should show the recipe name, cuisine & dietary restrictions (for filtering), preparation time (for sorting). There should also be a button to save or remove recipes from "My Recipe Book" depending on whether they have already been added.
